# Tether Web UI Project Setup - Complete Implementation Details

This document provides exhaustive, copy-paste-ready configuration for setting up the Tether Web UI project using Vite + React + TypeScript + shadcn/ui + Bun.

## 1. Exact Commands to Initialize Project

Execute these commands in order from the repository root (`/Users/jeffrey/code/tether`):

```bash
# Step 1: Create the Vite project with React TypeScript template
cd /Users/jeffrey/code/tether
bun create vite web-ui --template react-swc-ts

# Step 2: Navigate into the project
cd web-ui

# Step 3: Install base dependencies
bun install

# Step 4: Install Tailwind CSS v4 with Vite plugin
bun add -D tailwindcss @tailwindcss/vite

# Step 5: Install path resolution support
bun add -D @types/node

# Step 6: Install Biome for linting/formatting
bun add -D -E @biomejs/biome

# Step 7: Initialize Biome configuration
bunx @biomejs/biome init

# Step 8: Initialize shadcn/ui (interactive - select options below)
bunx shadcn@latest init
# When prompted:
#   - Style: new-york
#   - Base color: neutral
#   - CSS variables: yes

# Step 9: Install @hey-api/openapi-ts for API client generation
bun add -D -E @hey-api/openapi-ts

# Step 10: Install shadcn components (see Section 12 for full list)
bunx shadcn@latest add button card drawer tabs sonner skeleton badge separator scroll-area input label switch select dialog alert progress spinner
```

import path from "node:path";
import tailwindcss from "@tailwindcss/vite";
import react from "@vitejs/plugin-react-swc";
import { defineConfig, loadEnv } from "vite";

export default defineConfig(({ mode }) => {

  return {
    plugins: [react(), tailwindcss()],
    
    resolve: {
      alias: {
        "@": path.resolve(__dirname, "./src"),
      },
    },
    
    server: {

      },

    },

      outDir: "dist",
      sourcemap: false,
      minify: "esbuild",

## 12. List of shadcn Components to Install

Install all components with a single command:

```bash
bunx shadcn@latest add button card drawer tabs sonner skeleton badge separator scroll-area input label switch select dialog alert progress spinner
```

### Component Purpose Reference

| Component | Purpose in Tether |
|-----------|-------------------|
| `button` | Primary actions, form submissions |
| `card` | Dashboard widgets, settings panels |
| `drawer` | Mobile-first bottom sheets for settings/details |
| `tabs` | iOS-style bottom navigation, settings sections |
| `sonner` | Toast notifications for success/error feedback |
| `skeleton` | Loading states while fetching data |
| `badge` | Pass status indicators, connection status |
| `separator` | Visual dividers in lists and forms |
| `scroll-area` | Scrollable pass history, Wi-Fi network lists |
| `input` | Text inputs for Wi-Fi passwords, pass reasons |
| `label` | Form field labels |
| `switch` | Toggle settings (enable/disable features) |
| `select` | Timezone selection, network priority |
| `dialog` | Confirmation dialogs (use pass, restart system) |
| `alert` | Warning/info banners (low passes, connection issues) |
| `progress` | Signal strength visualization, onboarding progress |
| `spinner` | Loading indicators during async operations |

### `src/hooks/use-mobile.ts`

```typescript
import { useEffect, useState } from "react";

const MOBILE_BREAKPOINT = 768;

export function useIsMobile() {
  const [isMobile, setIsMobile] = useState<boolean | undefined>(undefined);

  useEffect(() => {
    const mql = window.matchMedia(`(max-width: ${MOBILE_BREAKPOINT - 1}px)`);
    const onChange = () => {
      setIsMobile(window.innerWidth < MOBILE_BREAKPOINT);
    };
    mql.addEventListener("change", onChange);
    setIsMobile(window.innerWidth < MOBILE_BREAKPOINT);
    return () => mql.removeEventListener("change", onChange);
  }, []);

  return !!isMobile;
}
```
